# Slack Send Skill

슬랙 메시지를 **권윤학님 어투로 다듬어 → 승인받고 → 전송/예약**하는 파이프라인.
`slack-messenger` role agent(작성·다듬기)와 Slack MCP(전송)를 잇는다.

> **불변 규칙: 승인 없이는 절대 전송하지 않는다.** 미리보기를 보여주고 사용자의 명시적 "OK"를 받은 뒤에만 `slack_send_message`/`slack_schedule_message`를 호출한다. 이건 이 스킬의 존재 이유다 — 잘못된 채널·오탈자·오발송 방지.

## 파이프라인

### 1. 입력 수집

요청에서 세 가지를 파악한다. 빠진 게 있으면 되묻는다(추측 금지).

- **내용**: 보낼 요점, 또는 다듬을 초안 원문
- **대상**: 채널(`#...`) 또는 DM 상대. 없으면 반드시 물어본다 — **대상 불명확 시 전송 단계로 진행하지 않는다.**
- **시각**: 언급 없으면 즉시 전송. "이따/내일 9시/오후 3시" 등이면 예약. 상대 시각은 KST(Asia/Seoul) 기준 절대 시각으로 환산해 확인한다.

### 2. 초안 작성 (slack-messenger)

`slack-messenger` role agent로 메시지를 작성하거나 다듬는다.

```
/agent slack-messenger "<대상 채널/상대 + 상황/요점, 또는 다듬을 초안>"
```

- 대상 채널로 레지스터(R1 정중 / R2 친근)를 판단하도록 채널 정보를 함께 넘긴다.
- 에이전트가 `[???]`로 남긴 빈 정보가 있으면 **여기서 채워 받는다** — 빈 채로 전송하지 않는다.

### 3. 대상 해소 (channel_id 확정)

- 채널: `slack_search_channels`로 이름 → `channel_id` 확인. 검색 결과가 여러 개면 **어느 채널인지 사용자에게 확인**한다(오발송 1순위 원인).
- DM: `slack_search_users`로 상대 → `user_id`(그대로 channel_id로 사용).
- 확정한 채널명·ID를 사용자에게 노출해 **대상이 맞는지 확인**한다.

### 4. 미리보기 + 승인 게이트 (필수)

아래를 한 화면에 모아 보여주고 명시적 승인을 받는다.

```
[보낼 곳] #team-프론트엔드 (C0XXXX)
[시각]    즉시 전송  /  또는  2026-07-20(월) 09:00 KST 예약
[내용]
<다듬어진 최종 메시지 전문>

이대로 보낼까요? (수정할 곳 있으면 말씀해주세요)
```

- 사용자가 "OK/보내/응" 등 **명시 승인**하기 전엔 5단계로 넘어가지 않는다.
- 수정 요청이 오면 2단계로 돌아가 다시 다듬고 재확인한다.
- 승인 문구가 모호하면("음..", "글쎄") 전송하지 말고 재확인한다.

### 5. 전송 / 예약

승인 후에만 호출한다.

- **즉시 전송**: `slack_send_message(channel_id, message)`
- **예약**: `slack_schedule_message(channel_id, message, post_at)` — `post_at`은 **2분 후 ~ 120일 이내** Unix timestamp(KST 환산). 범위 벗어나면 사용자에게 알리고 중단.
- 스레드 답글이면 `thread_ts` 지정.

### 6. 완료 보고

전송/예약 결과와 메시지 링크(permalink)를 사용자에게 돌려준다. 예약이면 "예약됨 + 발송 예정 시각"을 명시하고, 취소는 Slack UI의 "Drafts & sent"에서 가능함을 안내한다.

## Do-NOT

- **승인 전 전송 금지.** 어떤 경우에도 미리보기·승인을 건너뛰지 않는다.
- **대상 불명확 시 전송 금지.** 채널이 하나로 특정되지 않으면 물어본다.
- 사용자가 주지 않은 사실(수치·링크·담당자)을 지어내 채우지 않는다(`[???]`로 남기고 확인).
- Slack Connect(외부 공유) 채널에는 전송/예약 불가 — 걸리면 사용자에게 알린다.
- 대량 채널 동시 발송·반복 발송은 하지 않는다(요청이 명확해도 채널별로 확인).

## 에러 처리

| 상황 | 대응 |
|------|------|
| 채널 검색 0건/다수 | 후보 보여주고 사용자에게 선택 요청 |
| 예약 시각이 2분 미만/120일 초과 | 사용자에게 알리고 시각 재확인 |
| 전송 실패(권한/아카이브 채널) | 원인 보고, 재시도 여부 확인 |
| 승인 응답 모호 | 전송 보류, 명시 승인 재요청 |
